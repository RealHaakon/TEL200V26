\subsection{YuMi Application}
The control software for the dual-arm YuMi robot (IRB 14000) was developed in ABB's RAPID programming language using RobotStudio. The application implements a modular pick-and-place task that handles three distinct geometric objects --- cube, cylinder, and triangular prism --- triggered by external digital input signals. A central safety and recovery mechanism is provided through interrupt-driven emergency stop handling.

The program is contained in \texttt{MODULE Module1}. It defines a large number of \texttt{CONST robtarget} constants that store the taught positions for each object. These include safe approach points, gripping poses, lifting positions, transport poses, and drop-off locations. Each target is associated with the appropriate work object (\texttt{wobj}) --- \texttt{Kube}, \texttt{sylinder}, \texttt{prisme} for pick-up phases, and typically \texttt{wobj0} for home and global movements. The positions were taught using the FlexPendant or RobotStudio's interface and include complete pose data (Cartesian position, orientation quaternion, robot configuration, and external axes values).

The main entry point is the \texttt{Main()} procedure, which first initializes the gripper system by calling \texttt{g\_Init}. To avoid the common runtime error \texttt{ERR\_ALRDYCNT} caused by connecting the same interrupt multiple times, an initialization guard is used:

\begin{verbatim}
IF NOT interrupts_setup THEN
    SkipWarn;
    CONNECT stop_emer WITH EmergencyStop;
    ISignalDI di_EmergencySituation, high, stop_emer;
    interrupts_setup := TRUE;
ENDIF
\end{verbatim}

The program then enters an infinite loop that continuously polls three digital input signals (\texttt{di\_cube}, \texttt{di\_cylinder}, \texttt{di\_prism}) and calls the corresponding handling procedure when a signal is high:

\begin{verbatim}
WHILE TRUE DO
    IF di_cube = 1 THEN Cube;
    ENDIF
    IF di_cylinder = 1 THEN syl;
    ENDIF
    IF di_prism = 1 THEN prism;
    ENDIF
ENDWHILE
\end{verbatim}

Each object-specific procedure (\texttt{Cube}, \texttt{syl}, \texttt{prism}) follows a similar pick-and-place sequence:
\begin{enumerate}
    \item Move to home position
    \item Approach the pick-up area
    \item Open gripper (\texttt{g\_GripOut}) and wait
    \item Descend to gripping pose
    \item Close gripper (\texttt{g\_GripIn}) and wait
    \item Lift back to approach height
    \item Transport to drop area
    \item Descend to release pose
    \item Open gripper and wait
    \item Lift slightly
    \item Return to home position
\end{enumerate}

Most motions use \texttt{MoveJ} with speed \texttt{v500}  and accuracy \texttt{fine} at critical points or \texttt{z100} during faster transfers. 
Wait times of 2 seconds after gripper commands ensure reliable mechanical action.

The emergency stop is implemented as a trap routine triggered on the rising edge of \texttt{di\_EmergencySituation}:

\begin{verbatim}
TRAP EmergencyStop
    SkipWarn;
    StopMove;
    WaitUntil di_EmergencySituation = 0;
    StorePath;
    IF DInput(di_home) = 1 THEN
        ClearPath;
        MoveJ home, v500, fine, Servo\WObj:=wobj0;
        WaitTime 1;
        ExitCycle;
    ELSE
        RestoPath;
        StartMove;
    ENDIF
ENDTRAP
\end{verbatim}
When the emergency input becomes active, motion stops immediately (\texttt{StopMove}). 
After the signal is released, the program checks \texttt{di\_home}. 
If high, it clears the path context, moves to the safe \texttt{home} position, 
and executes \texttt{ExitCycle} to restart from the beginning of \texttt{Main()}. 
Otherwise it restores and resumes the interrupted path. This mechanism provides both immediate safety response and flexible recovery.

\subsection{YuMi Tic-Tac-Toe Challenge}
The Tic-Tac-Toe project was implemented on the dual-arm YuMi robot using two independent RAPID modules --- one for the left arm and one for the right arm. Each arm operates autonomously but follows the same general logic: to select and draw an ``X'' in one of nine grid cells based on incoming digital signals.

\subsubsection*{Program structure}
Both arms continuously poll three digital inputs:
\begin{itemize}
  \item \texttt{di\_cube}: increments the X coordinate
  \item \texttt{di\_cylinder}: increments the Y coordinate
  \item \texttt{di\_prism}: confirms execution
\end{itemize}
Two counters (\texttt{X} and \texttt{Y}) track the selected grid cell. Depending on the X--Y combination, the program calls one of several path procedures such as \texttt{Path\_P00}, \texttt{Path\_P11}, or \texttt{Path\_P22}.

The human--robot grid is divided between the arms as follows:

\subsubsection*{Path execution}
When a cell is selected, the corresponding path procedure executes a fixed sequence:
\begin{enumerate}
  \item Move to the home position.
  \item Move to \texttt{PenCenter} (safe pose above pen holder).
  \item Open gripper (\texttt{g\_GripOut}), descend to \texttt{PenPickDrop}, and close gripper (\texttt{g\_GripIn}) to pick up the pen.
  \item Move through a series of target points (\texttt{Target\_10}, \texttt{Target\_20}, etc.) to draw an ``X''.
  \item Return to \texttt{PenCenter}, drop off the pen, and return to home.
  \item Reset both \texttt{X} and \texttt{Y} to zero, ready for the next move.
\end{enumerate}
Joint motions (\texttt{MoveJ}) are used for most positioning, while linear motions (\texttt{MoveL}) ensure smooth, straight drawing lines. Speeds are set to \texttt{v500} with fine or small blending zones (\texttt{z50}--\texttt{z100}) depending on path precision.

\subsection{CAD Models}
To support reliable pen handling, a custom CAD design was created for both the \textbf{marker holder} and the \textbf{pen adapter}. The parts were designed in Autodesk Fusion 360 and 3D-printed in PETG.

The holder fixes the pen in a repeatable position on the table, while the adapter ensures proper orientation and grip alignment with the robot's fingers. This modular approach improved drawing consistency and made tool pickup or drop-off fully repeatable between runs.
Also a pen was designed in Autodesk that is identical in dimentions to the marker used in order to get correct point to floor distance.
